## Manual Read

`GBC06_01.png#16` is still the gating page-preview sample.

- `bt_lama_large` is still the better local branch for preserving the black
  diamond, but its page-level result is not final-quality because source
  structure remains visible and the rendered Chinese column is too narrow and
  stiff.
- `bt_aot` removes the original text more decisively, but the local sheet shows
  it harms the diamond area. That tradeoff is worse than LaMa for this sample.
- `gpt_image2_masked_edit` gets the strongest MIMO page score, but manual
  inspection rejects it as a direct production result: glyphs are noisy or
  overlapped, and using it as a cleanup crop before programmatic text rendering
  can create duplicate lettering.

`GBC06_01.png#17` is already usable enough for broader local-method expansion.

- `bt_lama_large` is the best current local result.
- `bt_aot` is close and remains a reasonable fallback/challenger.
- `gpt_image2_masked_edit` is visually useful as a reference, but does not beat
  LaMa for the final local pipeline.

## Decision

- Keep `bt_lama_large` as the current default non-bubble local inpainting path.
- Keep `bt_aot` integrated but do not promote it to default from this evidence.
- Keep `gpt-image-2` as experimental/reference-only for this stage. Its page
  result on `#16` is useful as a target, but it is not reliable final lettering.
- Treat MIMO page scores as evidence, not the final judge. In this run MIMO
  over-scored the `gpt-image-2` direct replacement relative to manual glyph
  quality.
